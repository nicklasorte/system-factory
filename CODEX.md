# Codex Instructions (repo)

- Read the relevant `*.system.yaml` before coding; honor `current_stage`.
- Keep scope to the requested stage and update the system file status when done.
- Avoid broad refactors; prefer small, reviewable diffs.
- Keep outputs deterministic and reproducible.
- Leave a clear handoff note: what changed, what remains, and the next recommended step.
- Run available validation/tests before opening a PR.
- If blocked, open a `blocked-decision` issue with options and proposed paths.
